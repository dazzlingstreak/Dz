元素类：Spot
操作类：Visitor

元素类可以接受不同的访问者来实现不同的操作，元素类中指定了操作的类型，具体的实现则由访问者实现。

优点：
使得增加新的访问操作变得很容易。
将有关元素对象的访问行为集中到一个访问者对象中，而不是分散到一个个的元素类中。
某一个访问方式有变化，只需要在具体的访问者类中修改。

缺点：
增加新的元素类很困难。新增一个元素时，则需要在访问者抽象类中新增访问方式，然后具体类各个都要改。修改一个原有的类的结构或者内部操作时，也会影响。
破坏封装。访问者模式要求访问者对象访问并调用每一个元素对象的操作，这意味着元素对象有时候必须暴露一些自己的内部操作和内部状态，否则无法供访问者访问

适用于元素类很少变化的情况。
有比较稳定的数据结构，又有易于变化的算法
